DocAtlas Feature List
=====================

Input & Extraction
- Supports .pdf, .doc, .docx, .ppt, .pptx, .xls, .xlsx
- Auto-unpacks .zip archives into a temporary staging area and processes supported files inside them
- Stores workbook-facing source paths relative to the selected input root
- Records unsupported files from normal folders and zip contents for intake reporting
- .doc auto-converted to .docx via LibreOffice (if installed)
- .xls auto-converted to .xlsx via LibreOffice (if installed)
- PDF extraction via pdfplumber
- OCR for PDFs: OCRmyPDF (non-forced pass, then forced pass only if needed) -> Tesseract fallback
- OCR of embedded images in .docx and .pptx when OCR is enabled
- Extracts Word tables and Excel sheets
- Extracts PowerPoint slide text

Summarization & Classification (LLM)
- Short summary (1â€“2 sentences)
- Long summary (5â€“7 sentences)
- Category assignment from your list
- Tag generation (deduped + capped)
- Optional article summaries for PDFs (disabled by default)
- If chat returns `content_filter`, a local extractive fallback summary is generated
- Documents using that fallback are flagged with `summary_fallback_content_filter`
- Very large documents use a deterministic local summary fallback and are flagged with `summary_truncated_large_doc`
- Final document category is selected deterministically from extracted text using tuned keyword/phrase rules

PDF Article Handling
- Disabled by default (opt-in via `--articles` in CLI or GUI toggle)
- When enabled, splits PDFs into articles using conservative strong-heading heuristics
- If structure is weak/ambiguous, defaults to one article to avoid over-splitting
- Stores article summaries separately when enabled

Duplicate Detection
- Exact duplicates: document byte hash (or normalized text hash for article rows)
- Near-duplicates: embeddings + cosine similarity (same-category, balanced rules + weak-edge structural guard)
- Embeddings source: full text (default), summary, or none

Outputs
- *_docatlas_summaries.xlsx
  - Documents (compact per-document review)
  - Duplicates (unified exact+near review groups)
  - Articles (conditional: written only when article generation is enabled; append mode preserves existing sheet)
- *_docatlas_import.xlsx (standalone import sheet for downstream KB import)
- Unreadable documents are excluded from import output by default
- *_docatlas_full_text.jsonl.gz (written by default; one JSON object per document, gzip-compressed)
- summary_report.txt
- unsupported_files_report.txt
- duplicate_groups_overview.xlsx inside each <category>_Duplicate folder
- resume.json (cache)
- last_run_stats.json (baseline for estimates)

Organization
- Moves files into category folders
- Unified duplicate review clusters (exact + near) into <category>_Duplicate/<ReviewGroupID>
- Auto-rename if file name conflicts
- Low/no-text files routed to Unreadable

GUI
- Folder pickers
- Application/category dropdown + editor
- OCR settings
- Embeddings source + append/overwrite toggle
- Charter Mode (preview-only; no file moves)
- Progress bar + ETA
- Pre-run runtime estimate popup (minutes/hours/days)

CLI
- Full CLI support for input/output/app/categories
- --workers N for parallel CLI processing (default 1)
- --articles to enable PDF article split/summarize/dedup (default is disabled)
- --no-articles kept as deprecated compatibility alias
- --category-path-map to supply import path mapping JSON
- Automatic pre-run validation ensures applications/categories in applications.json and category_path_map.json stay aligned
- --include-full-text-output kept as a deprecated compatibility flag; full-text archive is now always written
- --limit and --no-move for estimation runs
- --charter-mode (preview-only; no file moves)
- --overwrite-excel or append default
- --no-resume, --no-ocrmypdf
- --test-chat, --test-embeddings

Reporting & Cost
- Token estimates (chat + embeddings)
- No-text file list
- Extraction status counts
- File type breakdown and category percentages
- Unsupported file counts by datatype/source kind, plus top noisy source folders and a dedicated unsupported file inventory report
- OCR usage count and duplicate group stats
- Document length stats (avg/longest/shortest)
- Errors captured per file and reported without stopping the run
- Optional API call delay (DOCATLAS_API_DELAY) to reduce transient socket errors
- Retries for transient API/DNS/network failures with exponential backoff + jitter
  (DOCATLAS_API_MAX_RETRIES, DOCATLAS_API_RETRY_BASE, DOCATLAS_API_RETRY_MAX, DOCATLAS_API_TIMEOUT)

Hidden/Internal Features
- Auto-updated runtime baseline saved after every run
- Quick estimate uses file count + size + baseline
- Resume cache avoids reprocessing unchanged files
- Short-text guard skips embeddings for tiny inputs
- OCR logs note when OCR is triggered
- File key based on source path + mtime + size, or archive/member identity for zip contents, to detect changes
