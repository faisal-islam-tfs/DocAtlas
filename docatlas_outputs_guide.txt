DocAtlas Outputs Guide (Current)
================================

Core outputs per run
--------------------
DocAtlas writes:

1) <app_or_uncategorized>__docatlas_summaries.xlsx
   - Documents
   - Duplicates
   - Articles (conditional; see Articles sheet behavior below)

2) <app_or_uncategorized>__docatlas_import.xlsx
   - import
   - Columns:
     Id, Path, Title, Content, Summary, Tags, Attachments, AutoPublish, ArticleType

3) summary_report.txt

4) unsupported_files_report.txt

5) unsupported_cleanup.xlsx
   - cleanup_queue
   - cleanup_legend

6) docatlas.log

7) resume.json (if resume enabled)

8) last_run_stats.json

9) duplicate_groups_overview.xlsx
   - Created inside each <category>_Duplicate folder
   - Columns: Group ID, Relation, FileName, Exact_sc, Near_sc, Assigned to, Action

10) full_text_legacy_structure.txt
   - Historical note describing the retired Excel full-text workbook layout

Full-text archive behavior
--------------------------
- <app>__docatlas_full_text.jsonl.gz is written by default on every run.
- One gzip-compressed JSON object is written per document.
- Append mode dedupes by document identity (`file_key`) before adding new records.
- Overwrite mode rebuilds the archive from scratch.
- Legacy Excel structure remains documented historically in `full_text_legacy_structure.txt`.

Import sheet mapping behavior
-----------------------------
- Path uses category_path_map.json when available.
- Fallback path: <App>/<Category>
- ArticleType is content-based rule classification.
- AutoPublish is always true.
- `Unreadable` documents are excluded from the import workbook by default.
- `FilePath` values in review workbooks are relative to the selected input root.
- For files coming from archives, source references in workbooks use a logical relative path like `archive.zip!/inner/file.pdf` rather than the temporary staging path.

Unsupported file inventory
--------------------------
- `unsupported_files_report.txt` is always written per run.
- It records unsupported files found in normal folders and inside `.zip` archives.
- It also lists the top unsupported source folders for quick intake cleanup.
- When unsupported files exist, `unsupported_cleanup.xlsx` is also written as a human review queue.
- `cleanup_queue` columns:
  - `FileType`, `FileName`, `FilePath`, `SourceKind`, `SourceFolder`
  - `RecommendedAction`, `DeleteCandidate`, `Decision`, `DecisionNotes`, `ReviewedBy`, `FinalDisposition`
- `cleanup_legend` documents the supported decision values and row colors.
- Each line includes:
  - datatype / extension
  - file name
  - relative or logical source path
  - source kind (`file`, `zip_member`, `invalid_zip`)
- `summary_report.txt` also includes a compact unsupported-file summary:
  - total unsupported files
  - counts by datatype
  - counts from normal folders vs zip members vs invalid zips
  - top unsupported source folders

Articles sheet behavior
-----------------------
- Default is no-article mode: article generation is disabled unless explicitly enabled.
- Enable in CLI with `--articles` or via GUI article toggle.
- New/overwrite runs in no-article mode: no `Articles` sheet is created.
- Append runs in no-article mode:
  - if `Articles` already exists, it is preserved unchanged
  - if `Articles` does not exist, it is not created
- When article generation is enabled:
  - article splitting applies to PDFs only
  - splitting is conservative: multiple rows are created only with strong structural evidence
  - single-article PDFs produce one article row

File move behavior
------------------
- Non-duplicates: <category>/
- Unified review clusters (exact + near): <category>_Duplicate/<ReviewGroupID>/
- Grouping is same-category only.

Mode notes
----------
- Charter mode / no-move:
  - no file moves
  - duplicate folder overviews are still generated from review-group metadata
  - Excel/report outputs still generated
- Dry run:
  - no API calls, no moves
  - outputs still generated with available local extraction metadata
